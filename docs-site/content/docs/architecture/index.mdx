## Supporting Systems

- **Database** вҖ” SQLite via better-sqlite3, persists all state
- **UI** вҖ” Next.js 14 App Router with earthy matte theme
- **Integration** вҖ” Claude CLI wrapper, Git operations
- **Analytics** вҖ” Escalation patterns, improvement proposals, cost tracking

## Two-Document System

Flow's documentation follows a dual-document pattern:

| Doc Type | Contains |
|----------|----------|
| **Vision** (`docs/vision/`) | WHAT вҖ” purpose, behaviors, success criteria, status |
| **Design** (`docs/design/`) | HOW вҖ” implementation, APIs, file paths, code snippets |

Vision docs are stable (they capture intent), design docs evolve (implementation details change).

## Data Flow

1. **Input** вҖ” User submits request via CommandBar, CLI, or Telegram
2. **Classification** вҖ” Dispatcher routes to appropriate handler
3. **Planning** вҖ” Orchestrator creates capability and execution phases
4. **Building** вҖ” Skill/tool builders create capabilities
5. **Execution** вҖ” Ralph Workers claim and complete tasks
6. **Observation** вҖ” HOMR analyzes outputs, injects context
7. **Review** вҖ” Reviewer checks success criteria
8. **Learning** вҖ” Discoveries promoted to cross-outcome memory
